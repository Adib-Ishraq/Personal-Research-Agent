"""
RESEARCH AGENT WITH AUTOMATIC CLOUD UPDATES

This agent automatically keeps your research papers updated using GitHub Actions + Supabase.
It runs every day at 08:00 UTC without requiring your laptop to be on.

---

## SETUP (5 minutes)

### 1. Create Supabase Account (Free)
- Go to https://supabase.com
- Sign up with GitHub
- Create a new project
- Wait for it to be ready

### 2. Create Database Tables

In Supabase, open SQL Editor and run:

```sql
-- Papers table
CREATE TABLE papers (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  arxiv_id TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,
  summary TEXT,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Full-text search for papers
CREATE OR REPLACE FUNCTION search_papers(search_query TEXT, result_limit INT DEFAULT 10)
RETURNS TABLE (arxiv_id TEXT, title TEXT, summary TEXT, metadata JSONB) AS $$
BEGIN
  RETURN QUERY
  SELECT p.arxiv_id, p.title, p.summary, p.metadata
  FROM papers p
  WHERE p.title ILIKE '%' || search_query || '%'
     OR p.summary ILIKE '%' || search_query || '%'
  LIMIT result_limit;
END;
$$ LANGUAGE plpgsql;

-- Citation graph table
CREATE TABLE citation_graph (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  paper_id TEXT,
  parent_id TEXT,
  child_id TEXT,
  title TEXT,
  type TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Create indexes for performance
CREATE INDEX idx_papers_arxiv_id ON papers(arxiv_id);
CREATE INDEX idx_citation_graph_paper_id ON citation_graph(paper_id);
CREATE INDEX idx_citation_graph_parent ON citation_graph(parent_id);
```

### 3. Get Supabase Keys
- Go to Project Settings → API Keys
- Copy:
  - **Project URL** (supabase_url)
  - **anon public key** (supabase_key)

### 4. Create GitHub Repository
- Go to https://github.com/new
- Create repo: `research-agent`
- Clone: `git clone https://github.com/YOUR_USERNAME/research-agent.git`
- Copy all files from this project into the repo
- `git add .`
- `git commit -m "Initial research agent setup"`
- `git push`

### 5. Add GitHub Secrets
In your GitHub repo:
- Settings → Secrets and variables → Actions
- Add these secrets:
  - `GROQ_API_KEY` — Your Groq API key
  - `SUPABASE_URL` — From Supabase Project URL
  - `SUPABASE_KEY` — From Supabase anon public key
  - `TELEGRAM_BOT_TOKEN` — (optional, for notifications)
  - `TELEGRAM_CHAT_ID` — (optional, for notifications)

### 6. Test It
- Go to GitHub repo → Actions → Daily Research Auto-Update
- Click "Run workflow" → Run workflow
- Wait 1-2 minutes, check logs

---

## WHAT HAPPENS NOW

✅ **Every day at 08:00 UTC:**
1. GitHub Actions wakes up
2. Installs your dependencies
3. Searches ArXiv for papers on your topics
4. Processes new papers (fetches PDFs, summarizes with Groq LLM)
5. Chases citations recursively
6. Saves everything to Supabase
7. Sends you a Telegram notification with summary
8. Commits results to GitHub automatically

**You don't need to do anything — it works while you sleep!**

---

## LOCAL USE

To use locally (instead of cloud):

```bash
# Using local ChromaDB:
python main.py

# Using cloud (Supabase):
python main_cloud.py
```

---

## MONITORING

Check what the agent found:
1. **GitHub**: Settings → Actions → see all runs and logs
2. **Supabase**: Data Editor → browse papers and citations
3. **Telegram**: Get notifications for new papers
4. **Email**: GitHub emails you on workflow failures

---

## CUSTOMIZATION

Edit `topics.json` to change what the agent searches for:

```json
{
  "topics": [
    "your topic 1",
    "your topic 2",
    "your topic 3"
  ]
}
```

Commit and push — the agent will pick it up tomorrow.
